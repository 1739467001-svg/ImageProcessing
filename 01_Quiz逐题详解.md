# Quiz 1 — 光学 Optics

### Q1. 物高 0.01m，物距 u=0.20m，像距 v=0.40m。求焦距 f 和像高。
**公式** 1/f = 1/u + 1/v；m = v/u。
$$\frac1f=\frac1{0.20}+\frac1{0.40}=5+2.5=7.5\Rightarrow f=0.1333\,\text{m}=133.3\,\text{mm}$$
放大率 m = 0.40/0.20 = **2**。像高 = m × 0.01 = **0.02 m = 20 mm（倒立）**。

### Q2. 像素间距 10µm，512×512；物高 0.75m，物距 4m，要填满 90% 视场。求 f。
- 传感器高 = 512 × 10µm = **5.12 mm**；像高(填 90%) = 0.9 × 5.12 = **4.608 mm**。
- 放大率 m = 像高/物高 = 4.608mm / 750mm = **0.006144**。
- 像距 v = m·u = 0.006144 × 4 = 0.02458 m = **24.58 mm**。
- 焦距：1/f = 1/4 + 1/0.02458 → **f ≈ 24.4 mm**。
> 物体很远(4m)时 v≈f，所以 f≈像距≈24.4mm，符合直觉。

### Q3. f 数 N=2，物体能前后移动多少仍清晰（景深 Depth of Field）？
> ⚠️ 修正：讲义(ImageProcessing2 第31–34页)给了**精确的景深闭式公式**，要用这个，不是近似法。
弥散圆 C = 1 像素 = 10µm，N=2，u=4m，f≈24.43mm，f²=5.966×10⁻⁴，N·C·u = 2×10⁻⁵×4 = 8×10⁻⁵。
- **远界（可后移到）** S = f²u/(f² − N·C·u) = 2.386×10⁻³ / (5.966×10⁻⁴ − 8×10⁻⁵) = **4.62 m**（即向后 +0.62 m）。
- **近界（可前移到）** R = f²u/(f² + N·C·u) = 2.386×10⁻³ / (6.766×10⁻⁴) = **3.53 m**（即向前 −0.47 m）。
- **超焦距** H = f²/(N·C) = 5.966×10⁻⁴ / 8×10⁻⁵ = **7.46 m**（物体超过此距离则到无穷远都清晰）。
> 结论：物体可在 **3.53 m ~ 4.62 m**（即 **−0.47 m / +0.62 m**，总景深≈1.09 m）范围内移动仍清晰。注意景深**前后不对称**（远界比近界宽）。放大率小→景深大。

### Q4. 整个系统放进水中，对光学有何影响？
透镜功率由相对折射率决定：1/f=(n_lens/n_medium−1)(1/R₁−1/R₂)。
水的 n=1.33 > 空气 1.0 → **相对折射率减小 → 透镜变弱、焦距 f 变大、屈光度下降**。
后果：原来的成像面不再合焦，像距变长，需重新对焦/移动传感器。（玻璃 n≈1.5 时，功率约降到原来的 1/4，f 约变 4 倍。）

### Q5. 两透镜紧贴，P₁=40 D，P₂=10 D。求系统焦距。
紧贴 → 功率相加：P = 40+10 = **50 D**。f = 1/P = 1/50 = 0.02 m = **20 mm**。

### Q6. 两透镜 f₁=60mm、f₂=70mm，间距 d=200mm，物在 L1 前 200mm。求经 L2 后的像位置。
**逐镜计算：**
- L1：1/v₁ = 1/f₁ − 1/u₁ = 1/60 − 1/200 = 0.011667 → **v₁ = 85.71 mm**（L1 后方）。
- L1 的像作 L2 的物：u₂ = d − v₁ = 200 − 85.71 = **114.29 mm**。
- L2：1/v₂ = 1/f₂ − 1/u₂ = 1/70 − 1/114.29 = 0.005536 → **v₂ = 180.6 mm（L2 后方）**。
**答案：最终像在第二个透镜后方约 180.6 mm 处。**

### Q7. r=1.22λ/(2NA)，NA≈1/(2N)，λ=500nm，f=16mm，像素 1µm。求所需透镜直径 D。
- 化简：r = 1.22λ/(2·1/(2N)) = 1.22λN。
- 要分辨 → r = 像素 = 1µm：N = r/(1.22λ) = 1×10⁻⁶/(1.22×500×10⁻⁹) = **1.64**。
- D = f/N = 16mm / 1.64 = **9.76 mm**。

# Quiz 3 — 分割与颜色

### Q1. 用迭代法求阈值。直方图：强度1–7，像素数 [20,30,5,0,10,40,80]。
**迭代阈值法**（初始 T 取中点 (1+7)/2=4）：
- 组1 (≤4)：强度1,2,3,4，数 20,30,5,0 → μ₁=(1·20+2·30+3·5)/(55)=95/55=**1.73**。
- 组2 (>4)：强度5,6,7，数 10,40,80 → μ₂=(5·10+6·40+7·80)/130=850/130=**6.54**。
- 新 T=(1.73+6.54)/2=**4.13**。
- 再迭代分组不变 → **收敛于 T≈4.13**，即阈值取在 **4 与 5 之间**（实用取 T=4）。

### Q2. RGB={10,40,100} 转 YCrCb。
用本课 YUV 公式（U=Cb, V=Cr）：
- Y = 0.299·10 + 0.587·40 + 0.114·100 = 2.99+23.48+11.4 = **37.87**
  （若按课件笔误系数 0.229：Y=**37.17**）
- Cb = U = 128 − 0.169·10 − 0.331·40 + 0.5·100 = 128−1.69−13.24+50 = **163.07**
- Cr = V = 128 + 0.5·10 − 0.418·40 − 0.081·100 = 128+5−16.72−8.1 = **108.18**
**YCrCb ≈ (37.87, 108.18, 163.07)。**

### Q3. Y 用 6 bit、两个色度用 3 bit（范围 0–255）。求量化值及转回 RGB 的误差。
**量化步长**：6bit→2⁶=64 级，step=256/64=**4**；3bit→2³=8 级，step=256/8=**32**。
量化 = round(值/step)×step：
- Yq = round(37.87/4)×4 = 9×4 = **36**
- Crq = round(108.18/32)×32 = 3×32 = **96**
- Cbq = round(163.07/32)×32 = 5×32 = **160**
**转回 RGB**（解 YUV→RGB 线性方程组）：量化后恢复的 RGB ≈ (−8.95, 47.89, 92.68)，与原 (10,40,100) 比：
$$\text{误差}\approx(R:-18.95,\ G:+7.89,\ B:-7.32)$$
> 结论：色度只给 3 bit → 量化太粗，转回 RGB 后**误差很大**（R 甚至到负值需截断到 0），印证"色度可低精度但不能太低"。

### Q4. 写迭代灰度阈值的伪代码。
```
读入图像 img，初始化 T = 全图灰度均值
repeat
    G1 = { img 中所有 ≤ T 的像素 }
    G2 = { img 中所有 >  T 的像素 }
    μ1 = mean(G1);  μ2 = mean(G2)
    T_new = (μ1 + μ2) / 2
until |T_new − T| < ε       # 收敛
T = T_new
for 每个像素 p:              # 用最终 T 二值化
    if img(p) > T: out(p)=1 else out(p)=0
```

# Quiz 5 — 卷积与边缘检测

### 设置
10×10 图，rows 4–7 × cols 4–7 是 4×4 的白方块(=1)。沿 **row 5、cols 2–9** 这条线做卷积。
- Mask A（垂直边缘/水平梯度，**这其实是 Prewitt 核**；Sobel 是中间行 ±2 的加权版）：`[-1 0 1; -1 0 1; -1 0 1]`
- Mask B（拉普拉斯）：`[0 -1 0; -1 4 -1; 0 -1 0]`

### Q1. 沿该线的结果（cols 2→9）
> ⚠️ 修正：讲义(IP5 第24页)把梯度定义为 **Gx = f[i,j+1] − f[i,j]（右减左）**，这正是**相关(correlation)**的算法。所以**首选答案用相关**（与讲义梯度符号一致）：
**Mask A（相关，首选）**：`[0, +3, +3, 0, 0, −3, −3, 0]` → 左边界(col3–4)=+3、右边界(col7–8)=−3。
**Mask B（拉普拉斯，对称核，相关=卷积同值）**：`[0, −1, +1, 0, 0, +1, −1, 0]`

画成图（横轴 col 2–9，纵轴响应）：
- Mask A：在方块**左边界**出现 +3、**右边界**出现 −3，内部/外部都是 0 → 标出**两条竖直边**。
- Mask B：边界处出现 ±1 尖峰对，平坦区为 0 → 二阶导在边缘**过零/双峰**。
> 注：若题目要求**严格卷积(convolution)**（核翻转180°），Mask A 符号相反：`[0,−3,−3,0,0,+3,+3,0]`。**答题写清你用相关还是卷积**；讲义自己定义 Gx 时用的是相关式，所以默认按相关给分更稳。

### Q2. 对这两个操作的评论。
- **Mask A** 是**一阶梯度算子(Prewitt/Sobel 型)**，检测**垂直边缘**，对边缘给出有正负的强响应，平坦区为 0。
- **Mask B** 是**拉普拉斯(二阶导)**，各向同性，对边缘给**双极性(±)** 响应、对角点/噪声敏感；常用于锐化和过零检测。

### Q3. 若像素 (4,5) 被置 0（死像素），用什么非线性规则修复？
用**中值滤波**：取该像素 3×3 邻域的**中位数**替换它。邻域里大多是 1（方块内部），中值=1，于是被错误置 0 的像素被恢复为 1。非线性的中值不会像均值那样被异常 0 拉低。

# Quiz 6 — 描述子与 Hough

### Q1. 物体像素 (2,5)(3,6)(4,5)(4,7)(5,4)(6,3)，求主轴与 x 轴夹角 θ。
> 约定：第一个下标 i=行=垂直=y，第二个 j=列=水平=x。所以点 (x,y)：(5,2)(6,3)(5,4)(7,4)(4,5)(3,6)。

**质心**：x̄=(5+6+5+7+4+3)/6=30/6=**5**；ȳ=(2+3+4+4+5+6)/6=24/6=**4**。
**中心化** x́=x−5：[0,1,0,2,−1,−2]；ý=y−4：[−2,−1,0,0,1,2]。
**二阶矩（注意 b 有系数 2）**：
- a=Σx́² = 0+1+0+4+1+4 = **10**
- c=Σý² = 4+1+0+0+1+4 = **10**
- b=2·Σx́ý = 2·[0·(−2)+1·(−1)+0·0+2·0+(−1)·1+(−2)·2] = 2·(−6) = **−12**

**方向角（直接用带象限的反正切）**：
$$\theta=\tfrac{1}{2}\arctan2(b,\ a-c)=\tfrac{1}{2}\arctan2(-12,\ 0)=\tfrac{1}{2}(-90°)=\boxed{-45°\ (\equiv135°)}$$
**∴ 物体长轴(最小二阶矩轴)与 x 轴夹角 θ = −45°（即 135°）。**

**几何校验**：4 个点 (6,3)(5,4)(4,5)(3,6) 恰好落在直线 **x+y=9**（斜率 −1，即 −45°/135° 对角线）上 → 物体确实沿 −45° 方向伸长。✅ 与公式一致。

> ⚠️ **重要纠错（这章最大的坑）**：方向角**直接取** `θ=½·atan2(b, a−c)` 就是长轴方向，**不要再"代入 χ² 挑较小的那支"** —— 那样会挑到**垂直的短轴**(本题的 +45°)，挑反！
> 原因：课件 χ²=a cos²θ+b sinθcosθ+c sin²θ 里的 θ 其实是直线**法线**的角度（来自 ρ=x cosθ+y sinθ），所以 χ² 最小的 θ(=+45°) 是法线方向，真正的轴 = 法线 +90° = −45°。**直接用 ½·atan2(b,a−c) 最省事、最不会错。**
> 关键得分步骤：`b=2Σx́ý=−12`、`a=c=10` → `θ=½·atan2(−12,0)=−45°`。

### Q2. 四点 (−1,−1)(1,1)(2,2)(2,1) 在 Hough 空间的直线方程，指出不在主线上的点。
用极坐标式 **ρ = x cosθ + y sinθ**，每个点在 (ρ,θ) 空间是一条正弦曲线：
- (−1,−1)：ρ = −cosθ − sinθ
- (1,1)：ρ = cosθ + sinθ
- (2,2)：ρ = 2cosθ + 2sinθ
- (2,1)：ρ = 2cosθ + sinθ

**共线判断**：(−1,−1)(1,1)(2,2) 都在直线 **y=x** 上 → 该直线法线方向 θ=135°、ρ=0。
代入验证 θ=135°（cos135=−0.707, sin135=+0.707）：
- (1,1)：−0.707+0.707=**0** ✓；(2,2)：0 ✓；(−1,−1)：0 ✓ → 三条正弦曲线交于 **(ρ=0, θ=135°)**。
- (2,1)：2(−0.707)+1(0.707)=**−0.707 ≠ 0** ✗。
**∴ 点 (2,1) 不在主线上**（它的正弦曲线不过那个公共交点）。

# Quiz 7 — Conv2D（3 通道）

### 题. 3 通道 5×5 图，3×3×3 核，bias=0。求中心像素 Conv2D 输出及 ReLU。
中心像素 = 第(3,3)个，取每通道 rows2–4×cols2–4 的 3×3 窗口，与对应核**逐元素相乘求和**，三通道再相加。

**通道1**（核 `[-2,0,2;-1,2,1;1,-1,1]`，窗口 `[.2,.5,.8;.6,.1,.4;.2,.5,.8]`）：
行1：−0.4+0+1.6=1.2；行2：−0.6+0.2+0.4=0.0；行3：0.2−0.5+0.8=0.5 → **小计 1.7**
**通道2**（核 `[-4,0,4;-2,4,2;2,-2,2]`，窗口 `[.2,.7,.5;.6,.5,.8;.4,.1,.3]`）：
行1：−0.8+0+2.0=1.2；行2：−1.2+2.0+1.6=2.4；行3：0.8−0.2+0.6=1.2 → **小计 4.8**
**通道3**（核 `[2,0,-2;1,-2,-1;-1,1,-1]`，窗口 `[.2,.6,.3;0,.7,0;.9,.1,.9]`）：
行1：0.4+0−0.6=−0.2；行2：0−1.4+0=−1.4；行3：−0.9+0.1−0.9=−1.7 → **小计 −3.3**

**Conv2D 输出 = 1.7 + 4.8 − 3.3 + bias(0) = 3.2**
**ReLU(3.2) = max(0, 3.2) = 3.2。**

# Quiz 8 — 分类 / 池化 / BatchNorm

### Q1. 设计决策边界，把两个待测物体分到 A 类或 B 类。
观察训练数据：**A 类 Width 0.3–0.5、Length 1.0–1.4**；**B 类 Width 1.3–1.9、Length 4.5–8.0**。
Width 把两类分得最开 → 取边界 **Width = (0.5+1.3)/2 = 0.9**（或用 Length 边界 ≈ (1.4+4.5)/2 ≈ 3.0）。
决策规则：**Width < 0.9 → A 类，否则 → B 类。**
- 物体①：Width 0.2 < 0.9 → **A 类**（Length 1.1 也落在 A 区，一致）。
- 物体②：Width 1.6 > 0.9 → **B 类**（Length 4.0 也落在 B 区，一致）。
> 可写成线性判别 f = Width − 0.9：f<0 判 A，f>0 判 B。

### Q2. 4×4 激活，stride 2×2，求 Max/Average/GlobalAverage 池化。
```
0 1 0 4
0 2 1 0
3 0 1 1
2 1 1 2
```
四个 2×2 不重叠块：左上{0,1,0,2}、右上{0,4,1,0}、左下{3,0,2,1}、右下{1,1,1,2}。
- **MaxPooling** = `[[2, 4], [3, 2]]`
- **AveragePooling** = `[[0.75, 1.25], [1.5, 1.25]]`

### Q3. batch=2 的两张 2×2 图做 BatchNorm，γ=1.1，β=0.6，ε=1e−6。
image1 `[0, 0.4; 0.7, 0.7]`，image2 `[0.9, 0.5; 0.5, 0]`。单通道 → 8 个数一起算：
- 均值 μ = (0+0.4+0.7+0.7+0.9+0.5+0.5+0)/8 = 3.7/8 = **0.4625**
- 方差 σ² = Σ(x−μ)²/8 = **0.0923**，σ = **0.3039**
- 标准化并缩放平移 y = γ·(x−μ)/√(σ²+ε) + β：

| 原值 | x̂=(x−μ)/σ | y=1.1·x̂+0.6 |
|---|---|---|
| 0.0 | −1.522 | **−1.074** |
| 0.4 | −0.206 | **0.374** |
| 0.7 | 0.781 | **1.460** |
| 0.7 | 0.781 | **1.460** |
| 0.9 | 1.440 | **2.184** |
| 0.5 | 0.123 | **0.736** |
| 0.5 | 0.123 | **0.736** |
| 0.0 | −1.522 | **−1.074** |

**image1 BN ≈ [−1.074, 0.374; 1.460, 1.460]，image2 BN ≈ [2.184, 0.736; 0.736, −1.074]。**

# Quiz 9 — Viola-Jones

### Q1. 低功耗手机上为什么 Viola-Jones 适合做人脸定位？
1. **积分图**让任意矩形求和为 **O(1)**（只查 4 个角），不随窗口大小变慢；
2. **Haar 特征**仅是矩形和相减——**整数加减、无浮点无乘除**，省电省算力；
3. **级联分类器(cascade)**：前几级用极简特征**秒拒**绝大多数非人脸窗口，只有疑似区域才往下算 → 平均计算量极低，可在弱硬件上**实时**运行。

### Q2. 求 4×4 图的积分图。
原图：
```
0.2  0.2  0.6  0.4
0.3  0.3  0.6  0.3
0.6  0.7  0.9  0.1
0.1  0.2  0.15 0.2
```
积分图 II(i,j)=该点左上角所有像素之和（递推 II=i+上+左−左上）：
```
0.200  0.400  1.000  1.400
0.500  1.000  2.200  2.900
1.100  2.300  4.400  5.200
1.200  2.600  4.850  5.850
```
> 右下角 5.850 = 全图像素总和，可用来检查。

### Q3. 计算下列 Haar 特征值（忽略灰色）。Haar = 白区和 − 黑区和。
用 Q2 的像素值：
- **特征1（上白下黑，行划分）**：上两行(白) − 第三行(黑)，第四行灰忽略。
  白 = (0.2+0.2+0.6+0.4)+(0.3+0.3+0.6+0.3)=1.4+1.5=2.9；黑 = 0.6+0.7+0.9+0.1=2.3。
  **Haar = 2.9 − 2.3 = 0.6**
- **特征2（左白右黑，列划分）**：左两列(白) − 右两列(黑)。
  白=(0.2+0.2)+(0.3+0.3)+(0.6+0.7)+(0.1+0.2)=2.6；黑=(0.6+0.4)+(0.6+0.3)+(0.9+0.1)+(0.15+0.2)=3.25。
  **Haar = 2.6 − 3.25 = −0.65**
- **特征3（对角棋盘）**：白=左上块+右下块，黑=右上块+左下块。
  左上(rows1-2,cols1-2)=1.0；右下(rows3-4,cols3-4)=0.9+0.1+0.15+0.2=1.35；
  右上(rows1-2,cols3-4)=1.9；左下(rows3-4,cols1-2)=0.6+0.7+0.1+0.2=1.6。
  **Haar = (1.0+1.35) − (1.9+1.6) = 2.35 − 3.5 = −1.15**
> 约定：若老师定义 Haar=黑−白，符号取反即可，写清即可得分。**用积分图算这些矩形和可秒出**（这正是 Viola-Jones 快的原因）。
